class KShader{
public:
    static std::unique_ptr<KShader>  CreateFromFile(const std::string& filename, GLenum shaderType); 
    ~KShader(); 
    uint32_t Get() const { return m_shader; }
private:
    KShader(){} 
    bool LoadFile(const std::string& filename, GLenum shaderType ); 
    uint32_t m_shader { 0 }; 
}; 

< 설계패턴의 주요한 방식> : 
openGL자원을 사용해서 GPU에 메모리 누수를 막고 객체 생성의 성공여부를 제어하기 위한 방식으로 
스마트 포인터 사용을 강제 해서 혹여 delete등을 하지 않아서 자원이 낭비되는 것을 막는다. 

1.  정적 팩토리메서드 (Static Factory Methory) :객체 생성을 생성자가 아닌 CreateFromFile() 이라는 스태틱 메서드에서 함. 
    생성자는 반환값을 줄 수 없기에 객체 생성의 실패를 확인 할 수 없다. 반면 createFromFile는 nullptr을 리턴해서 
    실패을 null인지로 확인 할 수 있다. 

2. RAII (Resource Acquisition Is Initialization) 초기화와 동시에 자원 확보. - 게임엔진 설계에서 중요.
